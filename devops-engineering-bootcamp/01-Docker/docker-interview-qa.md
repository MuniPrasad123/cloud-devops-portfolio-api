# Docker Interview Q&A

## Interview-Ready Answers for Docker

These answers are written in a simple, professional style suitable for interviews, portfolio discussions, and mentoring conversations.

### 1. What problem does Docker solve?
Docker solves the problem of environment inconsistency. It packages an application and its dependencies into a portable image so the application can run the same way on a developer laptop, a test server, or a production machine.

### 2. Difference between Docker image and container?
A Docker image is a read-only blueprint. A container is a running instance of that image. The image is immutable, while the container can be started, stopped, and removed.

```text
Image = blueprint
Container = running instance
```

### 3. What is Dockerfile?
A Dockerfile is a text file that contains instructions for building a Docker image. It defines the base image, copied files, commands to run, and the startup command of the application.

### 4. What is Docker daemon?
The Docker daemon is the background service that manages Docker objects such as images, containers, networks, and volumes. The Docker CLI communicates with the daemon to perform actions.

### 5. What is Docker layer caching?
Docker layer caching means Docker reuses previously built layers when the same instructions are executed again. This makes builds faster because unchanged steps are not rebuilt.

### 6. Why Dockerfile order matters?
Dockerfile order matters because each instruction creates a layer. If frequently changing instructions are placed early, Docker invalidates later cache layers. Placing stable steps first and changing steps later improves caching efficiency.

### 7. Difference between docker stop and docker rm?
- `docker stop` stops a running container.
- `docker rm` removes a container from the system.

A stopped container can still be started again, but a removed container is deleted.

### 8. Difference between docker logs and docker exec?
- `docker logs` shows the output generated by a container.
- `docker exec` runs a command inside a running container.

Use logs to inspect runtime behavior and exec to debug the container filesystem.

### 9. What is Docker Hub?
Docker Hub is a registry where Docker images are stored and shared. It works like a central platform for distributing container images across teams and environments.

### 10. Difference between docker build, docker push, and docker pull?
- `docker build`: creates a local image from a Dockerfile.
- `docker push`: uploads an image to a registry such as Docker Hub.
- `docker pull`: downloads an image from a registry to your local machine.

### 11. Why does Docker say Layer already exists?
Docker says this when it finds that a layer already exists locally and can be reused from cache. It is generally a sign that the build is using cached layers efficiently.

### 12. Why Kubernetes/ECS needs a registry?
Kubernetes and ECS need a registry because they need a reliable place to pull container images during deployment. The registry stores the image layers and metadata so orchestrators can fetch them at runtime.

## Example for This Project

For the project `cloud-devops-portfolio-api`:
- Local image: `portfolio-api`
- Docker Hub image: `muniprasad07/portfolio-api:v1`
- Container examples: `portfolio-container`, `modest_lamport`
- App port: `8080`

Example flow:

```bash
docker build -t portfolio-api .
docker run -d --name portfolio-container -p 8080:8080 portfolio-api
docker tag portfolio-api muniprasad07/portfolio-api:v1
docker push muniprasad07/portfolio-api:v1
```
